**P04_PathFinding** Assignment Details =================== `Pathfinding` Implementation --------------------------- In the implementation of Pathfinder, the goal was to set the best path to take from point 'a' to point 'b'. For our implementation we created a board of PFNodes inorder to keep track of the paths cost and other functions. Another thing that we did was in commutePath was to make sure that as long as lastOnQ was not null we would push it onto the winning path. My group first worked on the easy functions like getPathStart, setPathEnd, etc. because they were pretty straight forward and easy to understand/ implement. Next we went step by step trying to figure out the things needed to be added to get the implementation to run correctly. We broke it up by function, trying to get that specific thing to work before moving forward to the next major function. `PFNode` Implementation ------------------------ In my group we implemented a stack of PFNodes. This would keep track of our winning path. Each PFNode comes with three different global variables. It has a currentLoc which tells you where you are. It has a previousLoc which tells you where you have been. Finally it has a cost that illustrates the cost of going to that specific location. These three things are all called when a user calls the constructor of a PFNode. The main function that we added to this was a setCost. This is a setter function and would set the cost equation to 'cost' otherwise if it was the first one the cost equaled zero. `Walker` Implementation ------------------------ The main idea for the walker function is to step through each coordinate on the winning path until reaching the end. We did this by writing a for loop that iterates through each coordinate on the winning path. For the advanced function we calculated the amount of time that it took to get from one coordinate to another and while that is greater than the time it needs to move forward it steps through the amount of coordinates while simultaneously subtracting the cumulative time by the time to move forward.
