Implementations =================== kd-Tree Node -------------- Our implementation of the kd-tree node was quite simple. Every node would hold a copy of a Point. This would allow us an easy way to get the x and y coordinates of the node. The node would also keep track of a value. This would allow us access to the value that is held at that particular x,y coordinate. Next we kept track of the left node and the right node because this allowed us to look at the next nodes left and right of the current. Finally we kept track of the partition of each node to see if it was an UPDOWN separation or a LEFTRIGHT separation. Nearest Neighbor ------------------ In our nearest function four main events happen. First, we check to see if the point passed in is equal to null. If it is a NullPointerException is thrown. Secondly, we check to see if the function isEmpty, then we return a null. Next we set the nearest node to be root and the nearestDist to equal the distance of the nearestNode's point distance. Finally we check to see if the nearestNode's point x is greater than the parameter p's x. If it is we recursively call our nearestR function first looking at nearestNode's left point and then looking at the nearestNode's right point. Otherwise we look at the opposite, first the nearestNode's right point and then the nearestNode's left point. Our helper function nearestR is where the nearest node is set and recursively looked for. It goes through a bunch of checks always looking at the left node first. If the function passes all of the recursive cases then the nearestDist is set to the current distance and the nearestNode is changed to be what is current.
